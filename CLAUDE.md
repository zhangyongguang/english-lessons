# Project: English error log (english-lessons)

Turn the Tencent Meeting transcripts of my daily 1-on-1 English class into a structured error log, used to review, find patterns, generate practice, and export to Anki.

## Key conventions
- **Raw transcripts** go in `data/raw/`, keeping the original Tencent filename; **the date is in the filename**: the first 8 digits of `..._20260528210658-...` are the class date (2026-05-28). Don't rename them.
- **Speakers**: `Jack` = me (the student), `ZIVA_Teacher` = the teacher. Only extract Jack's errors + the teacher's corrections.
- **A day may have several recordings**; merge them into one day's errors.
- **The fields and controlled tag vocabulary for each error** are in `templates/error_schema.md` ‚Äî follow them, don't invent synonym tags.
- **Errors are stored in two forms, in separate folders sharded by year-month**: `data/errors/json/YYYY-MM/DATE.json` (structured, feeds stats and Anki) + `data/errors/md/YYYY-MM/DATE.md` (readable, generated by a script). Paths are centralized in `scripts/_common.py` (`error_json_path` / `error_md_path` / `error_json_files`).
- **raw is never modified**; all processing produces new files.
- Transcripts have speech-recognition noise and mixed Chinese/English; restore my intended meaning from context. **The teacher's corrections are more reliable ‚Äî trust them first.**
- **Output language**: `explanation` / examples and the docs are in **English**; `my_sentence` / `correction` keep the original English. (Raw transcripts stay as recorded.)

## Commands (type `/` to invoke)
Error log (from class transcripts):
- `/extract [date|all]` ‚Äî extract errors ‚Üí `data/errors/json/YYYY-MM/DATE.json` + `data/errors/md/YYYY-MM/DATE.md`
- `/weekly [week]` ‚Äî weekly pattern summary ‚Üí `analysis/weekly/`
- `/exercise [tag]` ‚Äî generate targeted practice ‚Üí `exercises/generated/`
- `/anki` ‚Äî export error Anki cards ‚Üí `exercises/anki/anki_import.tsv`

Vocabulary (words I look up day to day):
- `/word [word(s)]` ‚Äî save looked-up words ‚Üí `data/vocab/vocab.json` + `vocab.md`. With arguments, saves those words; with none, harvests the words I asked about in the current chat session.
- `/vocab-quiz [topic]` ‚Äî quiz from recent/unmastered words ‚Üí `exercises/generated/`
- `/vocab-anki` ‚Äî export vocab Anki cards ‚Üí `exercises/anki/vocab_anki.tsv`

Misc:
- `/sync [message]` ‚Äî commit and push to GitHub (add + commit + push)

## How "new" is determined (important)
Not based on the system clock or file download time, but on **whether it has been processed**:
- **Day**: a transcript for some date exists in `data/raw/`, but `data/errors/json/YYYY-MM/DATE.json` does not ‚Üí that day is "pending". `/extract` by default handles the "most recent pending"; `/extract all` clears every pending day.
- **Week**: an ISO week has error data, but `analysis/weekly/YEAR-Wnn.md` does not exist ‚Üí that week is "pending". `/weekly` by default handles the "most recent pending" week.
- Benefit: which day you backfilled, several recordings in a day, or backfilling missed days later ‚Äî none of it matters; re-running won't reprocess (idempotent).
- Check status: `python3 scripts/list_raw.py` (by day), `python3 scripts/list_weeks.py` (by week).

## Vocabulary store (separate from the error log)
- A single growing file `data/vocab/vocab.json` (JSON array) is the source of truth; words are deduplicated by lowercase lemma, with `times_looked_up` bumped on re-lookup.
- Fields and controlled tags (`pos`, `topic`) are in `templates/vocab_schema.md`. Definitions and examples are in English; the first `example` must be one short, everyday sentence.
- `scripts/render_vocab_md.py` renders `vocab.md` as a `Word | Meaning | Example | Date` table, newest lookups first (Date = `first_seen`, last column since it's secondary; words still cluster by day for review). No per-day files and no master CSV ‚Äî a single file needs no merging.

## Scripts (pure standard library, no network or install needed)
- `scripts/list_raw.py` ‚Äî list transcripts, mark which days are pending
- `scripts/list_weeks.py` ‚Äî list errors by ISO week, mark which weeks are pending
- `scripts/render_md.py DATE` ‚Äî errors json ‚Üí readable md (both under data/errors/{json,md}/YYYY-MM/)
- `scripts/build_master.py` ‚Äî merge each day's JSON ‚Üí master CSV
- `scripts/make_anki.py` ‚Äî master table ‚Üí error Anki import file
- `scripts/render_vocab_md.py` ‚Äî vocab.json ‚Üí vocab.md (newest first)
- `scripts/make_vocab_anki.py` ‚Äî vocab.json ‚Üí vocab Anki import file
- `scripts/list_vocab.py` ‚Äî vocabulary status (totals, this week, repeat lookups)
- `scripts/validate.py` ‚Äî validate errors/vocab against the schemas (controlled vocab, required fields, duplicate id/word); read-only, used in CI
- `scripts/_common.py` ‚Äî shared helpers (date parsing, JSON loading, Markdown/TSV escaping); put common logic here, not duplicated
- Run with `python3` (use `python` on Windows).

## Tests & CI
- `tests/` holds a zero-dependency `unittest` suite (read-only ‚Äî it uses temp files, never touches real data). Run: `python3 -m unittest discover -s tests`.
- GitHub Actions (`.github/workflows/ci.yml`) runs the tests and `scripts/validate.py` on every push/PR.
- When changing a script that produces output, regenerate and confirm data files are untouched (`git status --short data/`).
