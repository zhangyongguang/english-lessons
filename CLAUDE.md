# Project: English error log (english-lessons)

Turn the Tencent Meeting transcripts of my daily 1-on-1 English class into a structured error log, used to review, find patterns, generate practice, and export to Anki.

## Key conventions
- **Raw transcripts** go in `data/raw/`, keeping the original Tencent filename; **the date is in the filename**: the first 8 digits of `..._20260528210658-...` are the class date (2026-05-28). Don't rename them.
- **Speakers**: `Jack` = me (the student), `ZIVA_Teacher` = the teacher. Only extract Jack's errors + the teacher's corrections.
- **A day may have several recordings**; merge them into one day's errors.
- **The fields and controlled tag vocabulary for each error** are in `templates/error_schema.md` — follow them, don't invent synonym tags.
- **Errors are stored in two forms**: `data/errors/DATE.json` (structured, feeds stats and Anki) + `DATE.md` (readable, generated by a script).
- **raw is never modified**; all processing produces new files.
- Transcripts have speech-recognition noise and mixed Chinese/English; restore my intended meaning from context. **The teacher's corrections are more reliable — trust them first.**
- **Output language**: `explanation` / examples and the docs are in **English**; `my_sentence` / `correction` keep the original English. (Raw transcripts stay as recorded.)

## Commands (type `/` to invoke)
- `/extract [date|all]` — extract errors → `data/errors/DATE.md` + `.json`
- `/weekly [week]` — weekly pattern summary → `analysis/weekly/`
- `/exercise [tag]` — generate targeted practice → `exercises/generated/`
- `/anki` — export Anki cards → `exercises/anki/anki_import.tsv`
- `/sync [message]` — commit and push to GitHub (add + commit + push)

## How "new" is determined (important)
Not based on the system clock or file download time, but on **whether it has been processed**:
- **Day**: a transcript for some date exists in `data/raw/`, but `data/errors/DATE.json` does not → that day is "pending". `/extract` by default handles the "most recent pending"; `/extract all` clears every pending day.
- **Week**: an ISO week has error data, but `analysis/weekly/YEAR-Wnn.md` does not exist → that week is "pending". `/weekly` by default handles the "most recent pending" week.
- Benefit: which day you backfilled, several recordings in a day, or backfilling missed days later — none of it matters; re-running won't reprocess (idempotent).
- Check status: `python3 scripts/list_raw.py` (by day), `python3 scripts/list_weeks.py` (by week).

## Scripts (pure standard library, no network or install needed)
- `scripts/list_raw.py` — list transcripts, mark which days are pending
- `scripts/list_weeks.py` — list errors by ISO week, mark which weeks are pending
- `scripts/render_md.py DATE` — DATE.json → readable DATE.md
- `scripts/build_master.py` — merge each day's JSON → master CSV
- `scripts/make_anki.py` — master table → Anki import file
- Run with `python3` (use `python` on Windows).
